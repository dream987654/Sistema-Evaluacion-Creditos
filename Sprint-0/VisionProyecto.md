# Visión del Proyecto
# Nombre del Proyecto

Sistema de Evaluación y Aprobación de Créditos para la Cooperativa Ética

# Descripción

El proyecto consiste en el diseño de un sistema de información que permita optimizar el proceso de evaluación y aprobación de créditos de la Cooperativa Ética. La propuesta busca reducir los tiempos de atención, mejorar el control de las solicitudes y facilitar el trabajo del personal encargado de la evaluación crediticia mediante una solución tecnológica basada en buenas prácticas de Ingeniería de Software.

# Problema

Actualmente, el proceso de evaluación de créditos presenta demoras debido al registro manual de información, dificultades en el seguimiento de las solicitudes y una limitada centralización de los datos, lo que puede ocasionar retrasos y errores durante la toma de decisiones.

# Objetivo General

Diseñar un Sistema de Evaluación y Aprobación de Créditos que optimice el proceso de evaluación crediticia mediante la aplicación de la metodología Scrum Adaptado y principios de Ingeniería de Software.

# Objetivos Específicos
Analizar el proceso actual de evaluación de créditos.
Definir los requisitos funcionales y no funcionales del sistema.
Diseñar la arquitectura de software del Sprint 3.
Elaborar los diagramas UML correspondientes.
Documentar el proyecto utilizando GitHub.

# Alcance
El proyecto contempla el análisis y diseño del sistema. Como parte del desarrollo técnico se trabajará específicamente el Sprint 3, correspondiente al módulo de Evaluación Crediticia, donde se desarrollarán la matriz de requisitos, casos de uso, arquitectura de software y diagramas UML.

# Metodología
Scrum Adaptado
UML
Arquitectura en Tres Capas
# Duración del Proyecto

4 meses
